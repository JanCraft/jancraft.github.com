Free web services

Do you want free web services? This article shows you how to host free web services in the cloud.


Front end
For hosting your static files you can use GitHub Pages, the only thing is that your static content will be publicly visible.
Another option is Firebase Hosting, which has a limit but keeps your source code private.

Back end (Serverfull)
There are 2 options for this one as well.
1) Cloudnode - Public source code and private environment variables (for secrets and API keys)
2) Heroku - Private source code but has a 15-20s delay on start and maximum monthly hours

There is also jDev's host, Denohost, but it relies on Heroku and has really slow start times (we're trying to improve that).

Back end (Serverless)
I don't really use serverless but here's two of them anyway:
1) Cloudflare Workers
2) Back4App

Conclusion
There's really lots of options, you can choose what you want to use for any project.
You can like anything, I have tried all of the services I have showed you (except Back4App) and have concluded you can use them all without any problems.

My preference
I use the GHPH (GitHub Pages + Heroku) stack but like a lot more the GHPC (GitHub Pages + Cloudnode) stack. I'm also going to try the GHPCW (GitHub Pages + Cloudflare Workers) stack.

Vercel / Now.sh / Zeit
Vercel combines static hosting and serverless in one free service, I have used it in the past and really like it, you can also use one of the many templates given to you by default.
